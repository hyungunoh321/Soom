# 숨표(SOOM) 배포 가이드

EAS(Expo Application Services)로 iOS/Android 앱을 빌드·배포하는 절차입니다.
`eas.json`(빌드 프로필)과 앱 식별자(`com.hyungunoh.soom`)는 이미 설정돼 있습니다.

## 1. 최초 1회 준비

```bash
# Expo 계정 로그인 (계정이 없으면 expo.dev에서 무료 가입)
npx eas-cli login

# EAS 프로젝트 생성·연결 (app.json에 projectId가 자동 추가됨)
npx eas-cli init
```

## 2. 빌드

| 명령 | 용도 |
|---|---|
| `npx eas-cli build --profile development --platform android` | 개발 빌드 (실기기에서 dev 서버 연결) |
| `npx eas-cli build --profile preview --platform android` | 내부 테스트용 APK (링크로 바로 설치) |
| `npx eas-cli build --profile production --platform all` | 스토어 제출용 |

- 처음 Android 빌드 시 키스토어 생성을 물어보면 **EAS 관리(권장)** 를 선택하면 됩니다.
- iOS 빌드에는 Apple Developer Program 계정($99/년)이 필요합니다.

## 3. 스토어 제출

```bash
npx eas-cli submit --platform android   # Google Play (개발자 계정 $25 1회)
npx eas-cli submit --platform ios       # App Store
```

## 4. 실제 지도 관련

- **iOS**: Apple 지도를 사용하므로 추가 설정이 필요 없습니다.
- **Android 스탠드얼론 빌드**: Google Maps API 키가 필요합니다.
  1. [Google Cloud Console](https://console.cloud.google.com)에서 프로젝트 생성 → *Maps SDK for Android* 활성화 → API 키 발급
  2. `app.json`의 `android`에 추가:
     ```json
     "config": { "googleMaps": { "apiKey": "발급받은 키" } }
     ```
  3. 다시 빌드
- **웹**: react-native-maps 미지원이라 기존 목업 지도로 자동 폴백됩니다.
- **Expo Go로 미리 보기**: `npx expo start` 후 실기기의 Expo Go 앱으로 QR 스캔 —
  API 키 없이도 실제 지도가 동작합니다 (iOS: Apple 지도 / Android: Expo Go 내장 키).

## 5. 백엔드 (Supabase)

- 클라이언트 키(`.env`의 anon key)는 공개용이라 빌드에 포함해도 안전합니다.
- 스키마 변경은 `supabase/schema.sql`을 수정한 뒤 대시보드 SQL Editor에서 실행하세요.
- 데모용으로 이메일 확인이 꺼져 있습니다. 실서비스 전에는
  Authentication → Sign In / Providers → Email → **Confirm email**을 다시 켜는 것을 권장합니다.
