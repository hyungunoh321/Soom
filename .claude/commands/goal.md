# /goal — 숨표 화면·기능 제작

## 목표
사용자가 지정한 화면 또는 기능을 **숨표 프로젝트의 IA와 디자인 시스템에 맞춰** 계획하고 구현한다.

**요청 대상:** $ARGUMENTS

## 진행 방식

1. **컨텍스트 확인**
   - `CLAUDE.md`의 디자인 토큰·IA·화면 목록·코딩 규칙을 먼저 읽는다.
   - 대상 화면에 해당하는 Screen ID를 IA 표에서 찾는다.
   - `references/` 폴더에 관련 시안 이미지가 있으면 참고한다.

2. **계획 제시 (구현 전 반드시)**
   - 만들 화면/기능, 사용할 Screen ID, 필요한 컴포넌트, 파일 위치, 사용할 디자인 토큰을 짧게 정리해 보여준다.
   - 새 라이브러리가 필요하면 여기서 함께 제안하고 확인을 받는다.
   - **확인을 받기 전에 코드를 작성하지 않는다.**

3. **구현**
   - 승인 후, 화면 하나 단위로 구현한다. 한 번에 여러 화면을 만들지 않는다.
   - 재사용 컴포넌트를 우선 활용하고, 없으면 새로 만들되 `components/`에 둔다.
   - 디자인 토큰은 하드코딩하지 말고 theme 상수에서 import 한다.
   - 하단 탭바 라벨은 홈·지도·저장·마이로 통일한다.

4. **검증**
   - 실제 실행 가능한 상태인지 확인한다(빌드/타입 에러 없음).
   - 구현한 내용과 다음에 만들면 좋을 화면을 한 줄로 안내한다.

## 원칙
- 불확실한 부분은 추측하지 말고 먼저 질문한다.
- 시안과 다르게 구현해야 할 이유가 있으면 솔직하게 말하고 이유를 설명한다.
