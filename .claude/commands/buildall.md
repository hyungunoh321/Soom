# /buildall — 숨표 앱 전체 완성

`CLAUDE.md`와 `docs/기획서.md`를 먼저 읽고, `references/` 폴더의 화면 시안 이미지들을 참고해서 숨표 앱 전체를 완성한다.

단, 아래 순서와 규칙을 반드시 지킨다.

## 1. 공통 세팅 (가장 먼저)
- 폴더 구조 정리
- 디자인 토큰: `constants/theme.ts` (CLAUDE.md의 컬러/스타일 토큰 기반)
- 공통 컴포넌트: `TagChip`, `SpotCard`, `CongestionBadge`, `StarRating`
- 하단 탭바 네비게이션: 홈 · 지도 · 저장 · 마이 (4개 라벨로 통일)

## 2. 화면 구현 (이 순서대로 하나씩)
각 화면을 끝낼 때마다 무엇을 만들었는지 한 줄로 보고하고 다음으로 넘어간다.

1. 홈 — SOOM_HOME_001
2. 지도 탐색 — SOOM_MAP_001
3. 장소 상세 — SOOM_SPOT_001, SOOM_SPOT_002, SOOM_SPOT_004
4. 후기 작성 — SOOM_SPOT_003
5. 저장/북마크 — SOOM_SAVE_001
6. 마이페이지 — SOOM_MY_001, SOOM_MY_002

## 3. 규칙
- 디자인 토큰은 하드코딩하지 말고 `theme`에서 import 한다.
- 각 화면은 해당 시안 이미지를 최대한 따른다.
- **지도 API 키, 로그인 방식처럼 결정이 필요한 부분은 멋대로 진행하지 말고 먼저 사용자에게 질문한다.**
- 화면 단위로 순서대로 진행한다. 전체를 뭉뚱그려 한 번에 쏟아내지 않는다.
- 모두 끝나면 실제 실행 가능한 상태인지 확인하고(빌드/타입 에러 없음), 실행 방법을 안내한다.
- 불확실하면 추측하지 말고 질문한다.
